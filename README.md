# ZJU-ECLab.github.io

The website of the **Emotion & Culture Lab, Zhejiang University** — including the
lab pages (Home, News, People, Publications, Resources, Courses, Join Us,
Contact) and the online edition of the weekly literature digest *《东西情报》*
(ECLab News).

Live at <https://zju-eclab.github.io>.

## Architecture

A static site generated by a small Python build script. You edit content as
Markdown / YAML; `build.py` renders it through Jinja2 templates into `dist/`,
which is published to GitHub Pages.

```
content/                 ← what you edit
  pages/                 prose pages (Markdown + YAML front-matter)
    home.md  join-us.md  contact.md  resources.md  resources/*.md
  news/                  one Markdown file per news post
  data/                  structured data for list pages
    people.yml  alumni.yml  publications.yml  courses.yml
templates/               Jinja2 templates (Material Design 3 layouts)
assets/                  shared CSS/JS, images, files
  tokens.css             shared M3 design tokens (lab + journal)
  site.css site.js       lab-site design system + interactions
  style.css app.js       the weekly-journal SPA (self-contained)
  logo.svg               lab mark
  img/ files/            migrated images and documents
journal/                 weekly-journal SPA (served at /journal/)
  index.html             SPA shell
  data/                  issue data (pushed by the ECLab-News pipeline)
build.py                 content + templates → dist/
flake.nix                dev shell with Python + jinja2/markdown/pyyaml
dist/                    generated output (git-ignored)
```

### Routes

| URL | Source |
|-----|--------|
| `/` | `content/pages/home.md` |
| `/news/`, `/news/<slug>/` | `content/news/*.md` |
| `/people/`, `/alumni/` | `content/data/people.yml`, `alumni.yml` |
| `/publications/` | `content/data/publications.yml` |
| `/resources/`, `/resources/<db>/` | `content/pages/resources*.md` |
| `/courses/`, `/join-us/`, `/contact/` | data / prose |
| `/journal/` | the weekly-journal SPA (`journal/index.html`) |

## Editing content

- **Prose pages & news**: edit the Markdown in `content/`. Front-matter sets the
  title, slug, etc.; the body is normal Markdown.
- **People / publications / courses / alumni**: edit the YAML in
  `content/data/`. In publications, wrap lab authors in `**bold**`.
- **Images / documents**: drop files into `assets/img/` or `assets/files/` and
  reference them with an absolute path (e.g. `/assets/img/people/name.jpg`).

## Build & preview

```bash
nix develop --command python3 build.py            # build into dist/
nix develop --command python3 build.py --serve    # build, then serve on :8000
```

(Without Nix: `pip install jinja2 markdown pyyaml`, then `python3 build.py`.)

The build fails loudly if any `/assets/...` reference has no matching file, so
broken images can't ship silently.

## The weekly journal

The journal is a self-contained SPA. Its issue data lives at `/journal/data/`
and is generated by the external [`ZJU-ECLab/ECLab-News`](https://github.com/ZJU-ECLab/ECLab-News)
pipeline, which pushes `journal/data/issues/<label>.json` and rebuilds
`journal/data/manifest.json` here. The deploy workflow rebuilds the site on every push
to `main`, so new issues go live automatically. That pipeline is untouched by
this site's build.

## Deployment

`.github/workflows/deploy.yml` builds with the pinned Nix toolchain and publishes
`dist/` to GitHub Pages on every push to `main` (and on manual dispatch). The
repo's **Settings → Pages → Source** must be set to **GitHub Actions**.

`.github/workflows/announce-issue.yml` is unchanged: it posts a Discussion and a
DingTalk notification when the pipeline adds a new issue.
